## 二、学习计划

### 2.1 计划创建

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 状态 |
|---------|---------|---------|-------|------|
| PL-001 | 智能生成计划 | 基于薄弱学科智能生成学习计划 | P0 | 待开发 |
| PL-002 | 手动创建计划 | 用户自定义计划名称、时间、任务 | P0 | 待开发 |
| PL-003 | 计划模板选择 | 提供期末冲刺/高考备战/日常复习等模板 | P1 | 待开发 |
| PL-004 | 计划周期设置 | 设置计划开始日期、结束日期 | P0 | 待开发 |
| PL-005 | 每日学习目标 | 设置每天的学习任务量 | P0 | 待开发 |

**计划类型**：

| 计划类型 | 说明 | 典型时长 |
|---------|------|---------|
| 日常计划 | 每日学习任务安排 | 持续进行 |
| 冲刺计划 | 考前集中复习 | 7-30天 |
| 专项计划 | 针对特定学科或章节 | 14-60天 |

**智能生成算法输入**：

| 参数 | 说明 |
|-----|------|
| 用户薄弱学科 | 正确率 < 60% 的学科 |
| 用户薄弱知识点 | 错误率 > 40% 的知识点 |
| 目标日期 | 计划结束日期 |
| 每日可用时间 | 用户设定的每日学习分钟数 |

**题目类型定义**：

| 题型ID | 题型名称 | 分值范围 | 是否客观题 |
|-------|---------|---------|-----------|
| 1 | 单选题 | 2-5分 | 是 |
| 2 | 多选题 | 3-8分 | 是 |
| 3 | 填空题 | 2-5分 | 部分 |
| 4 | 简答题 | 5-10分 | 否 |
| 5 | 计算题 | 8-15分 | 否 |

**难度等级定义**：

| 难度等级 | 名称 | 正确率范围 | 说明 |
|---------|------|-----------|------|
| 1 | 简单 | 70%-100% | 基础送分题 |
| 2 | 中等偏下 | 50%-70% | 需要一定理解 |
| 3 | 中等 | 30%-50% | 需要熟练掌握 |
| 4 | 中等偏上 | 15%-30% | 需要综合运用 |
| 5 | 困难 | 0%-15% | 选拔性题目 |

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 状态 |
|---------|---------|---------|-------|------|
| EX-010 | 智能组卷 | 设置条件后自动生成试卷 | P0 | 待开发 |
| EX-011 | 手动选题组卷 | 从题库勾选题目组成试卷 | P0 | 待开发 |
| EX-012 | 组卷条件设置 | 设置学科、难度、数量、时间 | P0 | 待开发 |
| EX-013 | 试卷预览 | 预览生成的试卷内容 | P0 | 待开发 |
| EX-014 | 试卷导出 | 导出为PDF格式 | P1 | 待开发 |
| EX-015 | 试卷保存 | 保存试卷模板便于重复使用 | P1 | 待开发 |

**智能组卷参数**：

| 参数 | 类型 | 说明 |
|-----|------|------|
| subject_ids | Array | 学科ID列表 |
| grade_level | Integer | 适用年级 |
| question_count | Integer | 总题数 |
| difficulty_distribution | Object | 难度分布 {easy: 30%, medium: 50%, hard: 20%} |
| knowledge_coverage | Float | 知识点覆盖率（0-1） |
| total_time | Integer | 建议时长（分钟） |
| total_score | Integer | 总分 |

**组卷算法说明**：

1. **知识点覆盖算法**：优先选择用户薄弱知识点，同时保证重点章节覆盖
2. **难度梯度算法**：按照简单-中等-困难的梯度分布设置题目
3. **去重算法**：避免与历史试卷重复率 > 30%
