## 1. 接口规范

### 1.1 基本规范

| 规范项 | 说明 |
|-------|------|
| 协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 请求方法 | GET（查询）、POST（创建）、PUT（更新）、DELETE（删除） |
| 认证方式 | JWT Bearer Token |

### 1.2 请求头规范

```
Content-Type: application/json
Authorization: Bearer {token}
X-Request-Id: {uuid}        # 请求追踪ID（可选）
X-Timestamp: {timestamp}     # 时间戳（可选）
```

### 1.3 分页参数

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| page | int | 否 | 页码，默认1 |
| pageSize | int | 否 | 每页条数，默认10，最大100 |

## 3. 错误码规范

### 3.1 错误码范围

| 范围 | 说明 |
|-----|------|
| 1000-1999 | 系统级错误 |
| 2000-2999 | 认证授权错误 |
| 3000-3999 | 业务参数错误 |
| 4000-4999 | 业务逻辑错误 |
| 5000-5999 | 资源相关错误 |

### 3.2 错误码定义

| 错误码 | 消息 | 说明 |
|-------|------|------|
| 1000 | 系统内部错误 | 服务器异常 |
| 1001 | 服务维护中 | 系统停机维护 |
| 1002 | 请求超时 | 接口响应超时 |
| 2000 | 未登录 | 需要登录 |
| 2001 | 登录已过期 | Token过期 |
| 2002 | 无效Token | Token格式错误 |
| 2003 | 无访问权限 | 权限不足 |
| 2004 | 账号已被禁用 | 用户状态异常 |
| 3000 | 参数不能为空 | 必填参数缺失 |
| 3001 | 参数格式错误 | 类型或格式不匹配 |
| 3002 | 参数超出范围 | 数值超出允许范围 |
| 4000 | 操作失败 | 业务操作失败 |
| 4001 | 资源不存在 | 数据不存在 |
| 4002 | 资源已存在 | 重复创建 |
| 4003 | 状态不允许 | 当前状态不支持该操作 |
| 4004 | 业务规则校验失败 | 不满足业务规则 |
| 5000 | 资源访问失败 | 文件/数据访问异常 |
| 5001 | 上传失败 | 文件上传异常 |
| 5002 | 下载失败 | 文件下载异常 |

## 4. 用户相关接口

### 4.1 用户注册

**请求信息**

```
POST /api/user/register
Content-Type: application/json
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| username | string | 是 | 用户名（6-20位字母数字） |
| password | string | 是 | 密码（8-20位） |
| role | string | 是 | 角色：student/parent |
| realName | string | 是 | 真实姓名 |
| phone | string | 否 | 手机号 |
| email | string | 否 | 邮箱 |
| grade | int | 否 | 年级（1=高一,2=高二,3=高三，role=student时必填） |

**请求示例**

```json
{
  "username": "student001",
  "password": "Pass123456",
  "role": "student",
  "realName": "张三",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "grade": 1
}
```

**响应示例**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "userId": 10001,
    "username": "student001",
    "role": "student",
    "bindCode": "123456",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

### 7.3 智能组卷

**请求信息**

```
POST /api/exam/paper/generate
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| subjectId | int | 是 | 学科ID |
| gradeLevel | int | 否 | 年级（默认用户年级） |
| totalScore | int | 否 | 总分（默认100） |
| totalMinutes | int | 否 | 建议时长（分钟，默认120） |
| strategy | string | 是 | 组卷策略 |
| difficulty | object | 否 | 难度分布 |
| chapterRatio | array | 否 | 章节比例 |

**strategy 选项**

| 值 | 说明 |
|---|------|
| intelligent | 智能组卷（按知识点覆盖） |
| weak | 薄弱专项（针对错题） |
| mock | 考前模拟（标准试卷结构） |
| chapter | 章节测试 |

**difficulty 参数**

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| easy | int | 否 | 简单题数量 |
| medium | int | 否 | 中等题数量 |
| hard | int | 否 | 困难题数量 |

**请求示例**

```json
{
  "subjectId": 1,

  "totalScore": 100,
  "totalMinutes": 120,

    "easy": 10,
    "medium": 8,
    "hard": 2
  },
  "chapterRatio": [
    {"chapterId": 11, "ratio": 0.4},

  ]
}
```
