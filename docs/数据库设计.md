## 2. 数据库命名规范

### 2.1 表命名规范

| 规则 | 说明 |
|-----|------|
| 单词数 | 1-3 个单词，使用下划线分隔 |
| 前缀 | 可选前缀，如 `sh_` (studyhelper) |
| 复数 | 使用复数形式（如 `users` 而非 `user`） |
| 统一性 | 同一模块的表使用相同前缀 |

### 2.2 字段命名规范

| 类型 | 规范 | 示例 |
|-----|------|------|
| 主键 | `id` 或 `{table}_id` | `id`, `user_id` |
| 外键 | `{referenced_table}_id` | `user_id`, `subject_id` |
| 时间 | `{action}_at` 或 `{action}_time` | `created_at`, `login_time` |
| 状态 | `status` | `status` |
| 布尔 | `is_{property}` 或 `{property}` | `is_active`, `is_deleted` |
| 排序 | `order` 或 `sort` | `order_num`, `sort_order` |
| 数量 | `{entity}_count` | `view_count`, `like_count` |

### 2.3 索引命名规范

| 类型 | 命名格式 |
|-----|---------|
| 普通索引 | `idx_{table}_{column}` |
| 唯一索引 | `uk_{table}_{column}` |
| 外键索引 | `fk_{table}_{column}` |
| 复合索引 | `idx_{table}_{col1}_{col2}` |

## 3. 核心表结构设计

### 3.1 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | 密码（BCrypt加密） |
| role | ENUM('student','parent','admin') | NOT NULL | 角色 |
| real_name | VARCHAR(50) | | 真实姓名 |
| phone | VARCHAR(20) | | 手机号 |
| email | VARCHAR(100) | | 邮箱 |
| grade | TINYINT | | 年级: 1=高一, 2=高二, 3=高三 |
| avatar_url | VARCHAR(255) | | 头像URL |
| bind_code | VARCHAR(6) | | 家长绑定码 |
| parent_user_id | INT | FK(users.id) | 关联家长ID |
| study_target | VARCHAR(50) | | 升学目标 |
| weak_subjects | VARCHAR(200) | | 薄弱学科(JSON数组) |
| admin_level | TINYINT | | 管理员级别 |
| status | TINYINT | DEFAULT 1 | 状态: 0=禁用, 1=正常 |
| last_login_at | DATETIME | | 最后登录时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

- `idx_username` (username)
- `idx_role` (role)
- `idx_bind_code` (bind_code)

## 4. 表关系说明

### 4.1 用户与角色关系
- `users.parent_user_id` → `users.id` (家长-学生关联)
- `users.role` 区分学生、家长、管理员

### 4.2 知识体系层级
- `subjects` → `chapters` (学科-章节)
- `chapters.parent_id` → `chapters.id` (章节-子章节)
- `chapters` → `knowledge_points` (章节-知识点)

### 4.3 题目与知识点
- `questions.knowledge_ids` JSON数组 → 关联多个 `knowledge_points`
- `questions.chapter_id` → `chapters.id`

### 4.4 试卷与题目
- `papers` → `paper_questions` → `questions` (多对多)

### 4.5 学习计划与任务
- `study_plans` → `daily_tasks` (一对多)
- `daily_tasks` → `subjects`, `chapters`, `knowledge_points`

### 4.6 学习追踪
- `users` → `study_records` → `knowledge_points`
- `users` → `user_favorites` → 知识点/题目/试卷
- `users` → `exam_records` → `papers`
- `users` → `wrong_questions` → `questions`
